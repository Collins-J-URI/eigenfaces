30  /** Hopefully occlusion-resistant subclass of PatternMatcher2D.
31  * This is not going to be the final version of this class. If this partial
32  * matching business works, I intend to create a MatchingParams class
33  * that would store all sorts of parameters specifying conditions for
34  * the match process (e.g., number of division of the partial match,, mask
35  * for occlusion by other tracked objects, etc.). So this one is just
36  * a quick coding with a hard-set 2x2 division into quadrants.
37  *
38  * As I was about half way through implementing this class, Meng and I
39  * did a lot of changes into the "regular" correlation matcher. The current
40  * coding of this class is a Q&D attempt at somehow matching the structure of
41  * the other class. I am not particularly fond of the difference between
42  * parameter signatures for RGBa and gray patterns. I also regret using the
43  * color as the first index of my expected values and covariance arrays
44  * [0=red, 1=green, 2=blue], because that means that the decomposition of the
45  * pattern into quadrants [5 indices: 0=whole, 1=UL, 2=UR, 3=LL, 4=LR] is
46  * explicitly stated in the API.
47  *
48  * @author jean-yves herve', 3D Group,
49  * Department of Computer Science and Statistics,
50  * University of Rhode Island
51  */
