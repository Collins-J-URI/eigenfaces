29  /** Image operator that computes differences between images.
30  *
31  * The BackgroundRemover_static class operates either in static mode (compute the
32  * difference between to raster images) or in "instance" mode" (compute the
33  * difference between an image and a "background" image stored by the
34  * BackgroundRemover_static object.
35  * <p>
36  * Note that image differences are always positive <code>unsigned char</code>
37  *
38  * The basic rules for image differences are as follows
39  * <ol>
40  * <li> The output image is a <code>RasterImage_gray</code> (possibly a
41  * <code>RasterImage_gray_F</code>);
42  * <li> If one of the two images has a "<code>float</code>" raster as reference,
43  * then that raster will be use to compute the difference;
44  * <li> If the difference function must <i>return</i> an image (i.e. the output
45  * image is not passed as a parameter), then if the difference function
46  * calculates a "<code>float</code>" difference, the raster returned will
47  * be of type <code>RasterImage_gray_F</code>;
48  * <li> If one of the two input images is a gray raster, the other image will be
49  * converted to gray before computing the difference;
50  * <li> If one image is RGB and the other RGBa, then the former will be converted
51  * to RGBA before computing the difference.
52  * </ol>
53  *
54  * <b>Very Important Note Regarding "Float" Images"</b><br>
55  * If an image has a float raster as reference, then that raster will be used to
56  * calculate the image difference. This can pose problems for float images that
57  * are not using the range 0-255. There are two workarounds for that problem:
58  * <ul>
59  * <li> rescale the raster's values to the desired range using
60  * <code>RasterImage::scaleValues(minVal, maxVal)</code>
61  * <li> call in sequence
62  * <ul><li><code>unsigned char** gray2D = myFloatImage->getGray2D()</code> (forces float to char conversion)
63  * <li><code>unsigned char** gray2D = myFloatImage->setGWorldAsReference(true)</code>
64  * </ul>
65  * </ul>
66  * Using the first solution, a <code>float</code> difference will be calculated while
67  * the second solution will give an "integer" difference.
68  *
69  * @author jyh and Fang Han
70  * @version 1.1
71  *
72  * (Abstract)Base class: BackgroundRemover
73  */
