Detailed Description

ElevationMap class An elevation map stores information relative to a surface of the form Z(X, Y), where X, Y, Z, are the coordinates of 3D points.

An ElevationMap object can be rendered as a raster image object or drawn as a 3D mesh.

An ElevationMap object stores an elevation map:

  • for a dense depth map, as a 2D raster Z[Y, X] (the row index of a raster corresponds to the Y coordinate, the column corresponds to X);
  • for a sparse depth map, it is assumed that the 3D points form an elevation surface along the Z direction

To be properly defined, the elevation map requires scaling parameters for the X, Y, and Z axes to transform raster indices into coordinates (the Z scaling parameter could in theory be dispensed with, since the elevation is normally stored in the float raster, but it is there for the sake of symmetry).

So far the only way to create a DepthMap, RangeMap, or ElevationMap object is to send it either a raster or a list of points to initialize it with. In other words, there is no "create now and allocate/initialize later" constructor.
