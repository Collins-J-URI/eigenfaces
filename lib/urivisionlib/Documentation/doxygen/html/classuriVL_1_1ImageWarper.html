Detailed Description

Virtual parent class for all image warper object.

Subclasses will implement a particular kind of warping algorithm (for example, by OpenGL-based texture mapping.

Author
Meng Wan, 3D Group for Interactive Visualization, Dept of Computer Science, University of Rhode Island
jean-yves herve', 3D Group for Interactive Visualization, Dept of Computer Science, University of Rhode Island

Computes the image dimensions that the warper can afford that are closest to the ones sent by the user.

Some warpers require specific image dimensions/ratios (e.g. a GL warper only takes 2^n x 2^n images). Furthermore, QuickTime GraphWorld rasters are padded on some platforms. The user should therefore first call imageEffectiveDimensions with desired image dimensions, and then create the image at the corresponding "effective" dimensions specified by that function.

If you want to warp an image, it is therefore recommended that you check what are the closest suitable dimensions for the type of warper you are using.

Parameters
imgWidthinput image width
imgHeightinput image height
lowImgWidthclosest lower suitable width
lowImgHeightclosest lower suitable height
