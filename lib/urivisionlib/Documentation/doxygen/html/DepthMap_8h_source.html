30  /** depth map class
31  * A depth map is a form of raster image that instead of storing an image itensity
32  * at each image point stores a depth value. A DepthMap object can be rendered as
33  * a raster image object or drawn as a 3D mesh.
34  *
35  * A DepthMap object stores an elevation map:
36  * <ul>
37  * <li> for a dense depth map, as a 2D raster Z[Y, X] (the row index of a raster
38  * corresponds to the Y coordinate, the column corresponds to X);
39  * <li> for a sparse depth map, it is assumed that the 3D points form an elevation
40  * surface along the Z direction
41  * </ul>
42  *
43  * The depth map (for me at least) is pretty much indisossiable from the notion of
44  * perspective projection. Therefore, all constructors for this class require an
45  * <code>IntrinsicParameters_pinhole</code> object (which will provide a focal
46  * length and scaling parameters for the inverse projection. If the camera used in
47  * orthographic, then the raster does not really store a depth map but
48  * an elevation map.
49  *
50  * So far the only way to create a <code>DepthMap</code>, <code>RangeMap</code>, or
51  * <code>ElevationMap</code> object is to send it either a raster or a list of
52  * points to initialize it with. In other words, there is no "create now and
53  * allocate/initialize later" constructor.
54  *
55  * @see ElevationMap
56  * @see RangeMap
57  */
