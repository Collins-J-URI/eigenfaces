Detailed Description

RangeMap class A range map stores information relative to a surface of the form Z(X, Y), where X, Y, Z, are the coordinates of 3D points.

An ElevationMap object can be rendered as a raster image object or drawn as a 3D mesh.

A RangeMap object stores a range map:

  • for a dense range map, as a 2D raster r[theta, phi] (the row index of the raster corresponds to the longitude theta, the column corresponds to the latitude phi);
  • for a sparse range map, it is assumed that the 3D points form a valid range surface (i.e. no two points are aligned with the origin)

A range map is what is produced by range finders such as scan lasers or radar. To be properly defined, the range map requires scaling parameters for the longitude, latitude, and range to transform raster indices into coordinates (the r scaling parameter could in theory be dispensed with, since the elevation is normally stored in the float raster, but it is there for the sake of symmetry)

So far the only way to create a DepthMap, RangeMap, or ElevationMap object is to send it either a raster or a list of points to initialize it with. In other words, there is no "create now and allocate/initialize later" constructor.
