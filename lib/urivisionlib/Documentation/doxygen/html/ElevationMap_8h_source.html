29  /** ElevationMap class
30  * An elevation map stores information relative to a surface of the form Z(X, Y),
31  * where X, Y, Z, are the coordinates of 3D points. An ElevationMap object can be
32  * rendered as a raster image object or drawn as a 3D mesh.
33  *
34  * An ElevationMap object stores an elevation map:
35  * <ul>
36  * <li> for a dense depth map, as a 2D raster Z[Y, X] (the row index of a raster
37  * corresponds to the Y coordinate, the column corresponds to X);
38  * <li> for a sparse depth map, it is assumed that the 3D points form an elevation
39  * surface along the Z direction
40  * </ul>
41  *
42  * To be properly defined, the elevation map requires scaling parameters for the
43  * X, Y, and Z axes to transform raster indices into coordinates (the Z scaling
44  * parameter could in theory be dispensed with, since the elevation is normally
45  * stored in the float raster, but it is there for the sake of symmetry).
46  *
47  * So far the only way to create a <code>DepthMap</code>, <code>RangeMap</code>, or
48  * <code>ElevationMap</code> object is to send it either a raster or a list of
49  * points to initialize it with. In other words, there is no "create now and
50  * allocate/initialize later" constructor.
51  *
52  * Version history:
53  * - v. 1.0 12/16/2004 [jyh]
54  *
55  * @author jean-yves herve', 3D Group,
56  * Department of Computer Science and Statistics,
57  * University of Rhode Island
58  */
