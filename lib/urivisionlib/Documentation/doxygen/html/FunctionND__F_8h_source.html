26  /** Pure virtual parent class for all n-dimensional functions.
27  *
28  * The function's actual implementation will be done in the child class and it will
29  * always be <strong>hard-coded</strong>.<p>
30  * Note that, contrary to a mathematical function, a <code>Function</code> is always
31  * defined on an <i>interval</i> that could be -&infin; to +&infin;, but cannot have
32  * "holes" in it.
33  *
34  * I have finally decided to remove all versions of func, gradient, and hessian that
35  * take (pointers to) ColumnVector and Matrix objects as parameters: Since all classes

37  * back and form between double and float.
