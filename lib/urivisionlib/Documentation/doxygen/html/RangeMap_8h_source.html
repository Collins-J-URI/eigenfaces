29  /** RangeMap class
30  * A range map stores information relative to a surface of the form Z(X, Y),
31  * where X, Y, Z, are the coordinates of 3D points. An ElevationMap object can be
32  * rendered as a raster image object or drawn as a 3D mesh.
33  *
34  * A RangeMap object stores a range map:
35  * <ul>
36  * <li> for a dense range map, as a 2D raster r[theta, phi] (the row index of the
37  * raster corresponds to the longitude theta, the column corresponds to the
38  * latitude phi);
39  * <li> for a sparse range map, it is assumed that the 3D points form a valid range
40  * surface (i.e. no two points are aligned with the origin)
41  * </ul>
42  *
43  * A range map is what is produced by range finders such as scan lasers or radar.
44  * To be properly defined, the range map requires scaling parameters for the
45  * longitude, latitude, and range to transform raster indices into coordinates
46  * (the r scaling parameter could in theory be dispensed with, since the elevation
47  * is normally stored in the float raster, but it is there for the sake of symmetry)
48  *
49  * So far the only way to create a <code>DepthMap</code>, <code>RangeMap</code>, or
50  * <code>ElevationMap</code> object is to send it either a raster or a list of
51  * points to initialize it with. In other words, there is no "create now and
52  * allocate/initialize later" constructor.
53  *
54  * - v. 1.0 12/17/2004 [jyh]
55  *
56  * @author jean-yves herve', 3D Group,
57  * Department of Computer Science and Statistics,
58  * University of Rhode Island
59  */
