23  /** Parent class for all descriptors of a camera's intrinsic parameters.
24  *
25  * A "camera" is a device (physical or virtual) that digitizes data to
26  * produce a raster. One could select to produce a raster for a small
27  * subpart of the available surface, but this would be a characteristic
28  * of the raster image produced, not of the camera.
29  *
30  * Since calculating the intrinsic parameters of a camera is an operation
31  * that is performed unfrequently, I have decided not to provide any
32  * "setVariable" method. The use desiring modify the value of a particular
33  * parameter should therefore create a new <code>IntrinsicParameter</code>
34  * object initialized with the proper values for all the parameters.<p>
35  *
36  * The basic geometric properties of a camera are the dimensions of the
37  * digitizable rectangle. Subclasses will add information relative to the
38  * optics of the camera (focal length, coordinates of the origin,
39  * distortion) and the details of the digitizing device (number and size
40  * of sensor elements).
41  *
42  * I have hesitated a long time about it, moving instance variables and functions
43  * relative to distortion and transformations between "pixel" and "world"
44  * coordinates back and forth between this class and its subclassses. I have finally
45  * decided to define at the root class the following functions:
46  * <ul>
47  * <li> <code>pixelToMetric</code>
48  * <li> <code>metricToPixel</code>
49  * <li> <code>distortMetric</code>
50  * <li> <code>distortPixel</code>
51  * <li> <code>undistortMetric</code>
52  * <li> <code>undistortPixel</code>
53  * </ul>
54  *
55  * Users of these functions should however be aware that
56  * <ul>
57  * <li> unless coordinates of the image center have been defined
58  * (<code>IntrinsicParameters_pinhole</code> and its subclasses),
59  * the image center is assumed to be at (0, 0).
60  * <li> unless dimensions of pixels have been defined
61  * (<code>IntrinsicParameters_CCD</code> and its subclasses),
62  * pixel and "metric" units are identical
63  * </ul>
64  *
65  * @author jean-yves herve', 3D Group,
66  * Department of Computer Science and Statistics,
67  * University of Rhode Island
68  */
